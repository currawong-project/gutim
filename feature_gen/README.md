

Using the group info file 'group_info.json' generated by the score pipeline
and the score tracked performance recordings from 
`score_follow_2/sf_study_8/.py` generate the performance
feature file `output/perf_feat.csv`.
`python main.py`


Using the score and preset feature files, `score_features.csv` and `preset_features.csv`
generated by the score pipeline, and the performance feature file `perf_feat.csv`
generated by `main.py` generate a set of training data.





Some proposed performance features:

- section_dur_sec: duration of the performance
- onset count:  count of onsets performed
- missed_pct:   percentage of missed notes.
- spurious_pct: percentage of spurious notes.
- time_factor:  tempo scaling factor
- vel_factor:   velocity scaling factor
- vel_delta:    mean deviation from scored velocity after fit
- vel_overall:  mean velocity
- grace_even:   mean std across all grace groups
- TODO: grace_note_dur: mean of grace note dur (w/o pedal)     
- TODO: grace_note_std: std of grace note dur (w/o pedal)      
- grace_grp_dur:mean grace group duration deviation pct
- beat_even:    mean std across all beat groups
- dyn_acc:      mean delta of velocity when fit to dlevel.
- chord_spread: mean chord time spread
- tempo:        measured tempo in beats_per_quarter

- avg/med/std of raw key-velocity (rt)
- avg/med/std abs(key-velocity - score-velocity) (rt)
- avg/med/std of time between onsets (rt)
- avg/med/std abs(onset-time - score-time) (rt)
- avg/med/std tempo (where the tempo is measured at each matched note) (rt)
- avg/med/std abs(beat-onset-time - beat-score-time)  (rt)
- avg/med/std grace-group duration (rt)
- avg/med/std chord onset spread (rt)
